# Consumer Flux overlay — template

Copy this directory into your consumer repo at a per-env path that
matches your existing layout. Example:

```bash
cp -r examples/consumer/flux/ ~/path/to/my-consumer-repo/environments/<env>/flux/
```

Then:

1. Replace every `CHANGE-ME` placeholder with your env's values.
2. Pin OCD to a release tag in `platform-overlay/kustomization.yaml`
   (the `?ref=` query string on the remote-base URL).
3. Apply `examples/consumer/06-flux-bootstrap` TF layer with
   `env_name = <your-env>` and `github_owner/repo` pointing at your
   consumer repo.
4. After Flux + the sealed-secrets controller are up, run
   `scripts/init-flux.sh --consumer-dir <your-env-flux-dir>` to seal
   your secrets against the controller and commit them.

## What each file does

- `sources.yaml` — Flux `GitRepository` resources pointing at OCD
  (pinned tag) and at this consumer repo. Two sources because the
  consumer repo provides the overlay + the secrets; OCD provides the
  shared platform + infrastructure.
- `infrastructure.yaml` — Flux `Kustomization` that reconciles
  `flux/infrastructure/` from the OCD source. No overlay — every env
  gets the same shared add-ons.
- `platform.yaml` — Flux `Kustomization` that reconciles
  `./platform-overlay/` from the consumer repo. The local
  `platform-overlay/kustomization.yaml` references OCD's
  `flux/platform/` as a remote Kustomize base and adds the consumer's
  patches + sealed secrets.
- `platform-overlay/kustomization.yaml` — the Kustomize file that does
  the actual overlay work.
- `sealed-*.yaml` — sealed secrets generated by the init wizard (or by
  hand with `kubeseal`).
